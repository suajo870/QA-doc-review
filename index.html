GitHub Pages 배포 가이드

이 사이트를 팀원 모두가 접속할 수 있게 배포하는 방법

🚀 배포 순서 (약 10분 소요)
1

GitHub 계정 만들기

github.com/signup 에서 무료 계정을 생성하세요. (팀 공용 계정 추천)

2

새 Repository(저장소) 만들기

로그인 후 우측 상단 + → New repository 클릭
Repository name: qa-doc-review
Visibility: Public 선택 → Create repository

3

index.html 파일 업로드

다운로드한 index.html 파일을 Repository 페이지로 드래그하거나,
uploading an existing file 링크를 클릭해 업로드하세요.

4

GitHub Pages 활성화

Repository → Settings → 좌측 Pages 메뉴
Source: Deploy from a branch
Branch: main / / (root) 선택 → Save

5

접속 URL 공유

1~2분 후 아래 주소로 접속 가능해요. 팀원들에게 이 링크를 공유하세요.

https://[GitHub계정명].github.io/qa-doc-review
⚠️ 데이터 공유 주의: 현재는 각 팀원의 브라우저에 개별 저장됩니다. 팀원 간 데이터 공유가 필요하면 추후 Google Sheets 연동을 추가해드릴 수 있어요.
